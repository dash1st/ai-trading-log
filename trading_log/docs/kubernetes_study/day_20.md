20일차: 클러스터의 기억 장치, etcd
1. etcd란 무엇인가요?
쿠버네티스의
모든 상태 정보(설정, 상태, 권한 등)를 저장하는 Key-Value 형태의 데이터베이스
입니다. 쿠버네티스의 '뇌' 또는 '기억 장치'라고 부르기도 합니다.
우리가
kubectl apply
로 명령을 내리면, 그 결과값은 결국 이 etcd에 저장됩니다. 만약 etcd가 사라지면 쿠버네티스 클러스터는 치매에 걸린 것처럼 아무것도 기억하지 못하고 멈춰버립니다.
2. 보안 전문가를 위한 etcd 체크포인트
15년 차 보안 전문가이신 사용자님께 etcd는 **'최우선 보호 대상(Top Priority)'**입니다.
데이터 암호화 (Encryption at Rest):
etcd 안에는 시크릿(Secret) 같은 민감한 정보가 저장됩니다. 기본적으로는 평문으로 저장되므로, 반드시 암호화 설정을 활성화해야 합니다.
접근 통제:
etcd는 API 서버와만 통신해야 합니다. 외부나 다른 파드에서 etcd 포트(2379)에 직접 접근하는 것은 보안상 치명적입니다.
고가용성 (HA):
데이터가 유실되면 끝입니다. 보통 3개 이상의 홀수 개로 클러스터를 구성하여 데이터의 안정성을 보장합니다.
3. 백업과 복구
보안의 마지막 보루는 백업이죠.
etcdctl snapshot
명령어를 통해 정기적으로 상태를 저장해두어야 사고 발생 시 클러스터를 원래대로 되돌릴 수 있습니다.
4. 쉬운 비유
쿠버네티스 클러스터:
거대한
'정부 조직'
API 서버:
민원을 접수하는
'창구 직원'
etcd:
모든 법전과 기록물이 보관된
'중앙 문서고'
. 문서고가 불타버리면 정부 시스템 전체가 마비되는 것과 같습니다.
💡 오늘의 요약
"etcd는 쿠버네티스의 모든 기록이 담긴 '중앙 데이터베이스'다. 이곳이 뚫리면 클러스터 전체 권한이 넘어가는 것과 같으므로, 강력한 암호화와 철저한 접근 통제가 필수적이다!"
실무 활용 팁:
보안 점검 시
etcd
가 TLS 통신을 강제하고 있는지, 그리고 백업 데이터가 오프사이트(Off-site)에 안전하게 보관되고 있는지 확인하는 것이 핵심 점검 항목입니다.
