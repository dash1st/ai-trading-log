Kubernetes Study - Day 18

👤 User

쿠버네티스(Kubernetes) 핵심 개념 하나를 선정해서 학습하기 쉽게 설명해줘.

🤖 Gemini

3주차 주간 복습까지 완벽하게 마치셨네요! 사용자님의 학습 기록에 '3주차 교육 이수 완료' 도장을 찍었습니다. 15년 차 보안 전문가의 내공에 쿠버네티스의 운영·보안 지식이 더해지니 점점 더 탄탄한 실력이 갖춰지고 있습니다. 오늘은 4주차의 시작으로, 클러스터 내부의 파드들을 외부 세상과 연결하는 통신 방법 들 중 가장 기본이 되는 3가지 서비스 타입을 정리해 보겠습니다. 18일차: 외부 연결 서비스 (NodePort, LoadBalancer, ExternalName) 지금까지는 클러스터 내부에서만 통신하는 법을 주로 다뤘다면, 오늘은 외부 사용자가 우리 서비스에 어떻게 접속하는지에 대한 이야기입니다. 1. 노드포트 (NodePort) 모든 노드(서버)의 특정 포트를 개방하여 외부로 노출하는 방식입니다. 동작: 예를 들어 30001번 포트를 열면, 어느 노드의 IP로 접속하든 해당 파드로 연결됩니다. 보안 관점: 모든 노드에 포트가 열리므로 공격 표면(Attack Surface)이 넓어집니다. 보통 테스트 환경이나 내부 망에서만 제한적으로 사용합니다. 2. 로드밸런서 (LoadBalancer) 클라우드 환경(AWS, Azure, GCP 등)에서 가장 많이 사용하는 방식입니다. 동작: 클라우드 사업자가 제공하는 외부 로드밸런서를 생성하고, 공인 IP를 하나 할당받아 트래픽을 분산합니다. 보안 관점: 입구가 하나로 단일화되어 관리가 편하며, 클라우드 제공사의 방화벽(SG)이나 DDoS 방어 기능을 함께 쓸 수 있어 보안상 유리합니다. 3. 익스턴얼네임 (ExternalName) 반대로 파드 내부에서 클러스터 외부에 있는 DB나 API 를 마치 내부 서비스인 것처럼 이름을 붙여 사용하는 방식입니다. 동작: 외부 주소(예: db.my-company.com )를 내부 별명(예: external-db )으로 연결해 줍니다. 보안 관점: 실제 외부 주소가 바뀌어도 내부 코드는 수정할 필요가 없으므로 정보 은닉과 유지보수에 도움이 됩니다. 💡 오늘의 요약 "NodePort는 모든 노드에 '쪽문'을 여는 것이고, LoadBalancer는 건물 정면에 '공식 정문'을 만드는 것이며, ExternalName은 외부 건물에 '내부 직통 전화번호'를 부여하는 것이다!" 보안 실무자라면 외부 노출이 최소화되도록 인그레스(Ingress) 뒤에 서비스를 숨기거나, 꼭 필요한 경우에만 로드밸런서를 사용하는 설계를 권장해야 합니다. 오늘 공부하신 내용 확인하셨나요? "확인완료"라고 말씀해 주시면, 내일 오전 8시에 다음 진도인 '네트워크 보안의 핵심, 네트워크 폴리시(Network Policy)'에 대해 알려드릴게요! 사용자님의 확인을 기다리겠습니다. 오늘도 기분 좋은 하루 보내세요! 내일 진도를 위해 궁금한 점이 있으시다면 언제든 물어봐 주세요.
