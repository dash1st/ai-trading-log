32일차: 클러스터의 블랙박스, 감사 로그(Audit Log)
1. 감사 로그란 무엇인가요?
쿠버네티스 API 서버를 거쳐가는 모든 요청(누가, 언제, 어디서, 무엇을 했는지)을 기록하는
최상위 보안 기록
입니다.
15년 차 보안 전문가이신 사용자님께 이 로그는 단순한 기록이 아니라, 침해 사고 발생 시 해커의 경로를 파악할 수 있는 **'결정적 증거(Digital Forensics)'**입니다.
2. 로그에 기록되는 4단계 (Stages)
하나의 요청은 다음 단계별로 기록될 수 있습니다.
RequestReceived:
요청이 들어온 즉시 기록.
ResponseStarted:
응답 헤더가 보내진 시점. (긴 응답의 경우 유용)
ResponseComplete:
요청 처리가 완벽히 끝난 시점. (가장 많이 사용)
Panic:
요청 처리 중 내부 에러가 발생했을 때.
3. 보안 전문가를 위한 '감사 정책' 팁
모든 로그를 다 기록하면 용량이 너무 커집니다. 따라서 다음과 같이 차등 적용합니다.
Secret/ConfigMap:
어떤 데이터가 오갔는지(RequestResponse) 상세히 기록하여 유출 여부 감시.
Pod/Deployment:
누가 생성/삭제했는지(Metadata) 위주로 기록.
시스템 컴포넌트:
안정성을 위해 로그 수준을 낮춤(None).
4. 쉬운 비유
쿠버네티스 API 서버:
정부 청사의
'민원 창구'
감사 로그:
민원실에 설치된
'고화질 CCTV와 출입 명부'
.
"누가 몇 시에 들어와서 어떤 서류를 열람했고, 직원이 승인해 줬는지"를 한 글자도 빠짐없이 기록하는 보안 대장과 같습니다.
💡 오늘의 요약
"감사 로그는 클러스터의 '최후의 보루'다. 인드레스 방화벽이 뚫리더라도, 감사 로그가 살아있다면 해커가 어떤 권한을 탈취해 무슨 짓을 했는지 완벽하게 복기할 수 있다!"
실무 활용 팁:
감사 로그는 클러스터 내부가 아닌, 지난번에 배운
EFK 스택
이나 외부
SIEM(보안 정보 이벤트 관리)
장비로 즉시 전송하여 변조되지 않도록 격리 보관하는 것이 핵심입니다.
