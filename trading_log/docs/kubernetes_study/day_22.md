# Kubernetes Study - Day 22

## 👤 User
쿠버네티스(Kubernetes) 핵심 개념 하나를 선정해서 학습하기 쉽게 설명해줘.

## 🤖 Gemini
어제까지 **이미지 보안(공급망 보안)**을 통해 '깨끗한 부품'을 들여오는 법을 배우셨죠? 하지만 아무리 깨끗한 이미지를 썼더라도, 실행 중인 컨테이너에 해커가 침투해 악성 활동을 할 수도 있습니다.
그래서 오늘은
22일 차
, 실행 중인 파드 내부의 수상한 움직임을 실시간으로 잡아내는 **'런타임 보안과 팔코(Falco)'**에 대해 알아보겠습니다.
22일차: 런타임 보안의 파수꾼, 팔코(Falco)
1. 런타임 보안이 왜 필요한가요?
보안 전문가이신 사용자님께는
'사후 탐지 및 대응'
영역입니다. 이미 배포된 파드 내부에서 갑자기 쉘(Shell)이 실행되거나, 중요 설정 파일(/etc/shadow 등)이 수정된다면? 이는 침해 사고의 강력한 증거입니다. 이를 실시간으로 감시하는 도구가 필요합니다.
2. 팔코(Falco)란 무엇인가요?
팔코는 쿠버네티스의 **실시간 침입 탐지 시스템(IDS)**입니다. 리눅스 커널 수준에서 발생하는 이벤트를 가로채서, 미리 정해둔 보안 규칙에 어긋나는 행동이 감지되면 즉시 알람을 울립니다.
3. 팔코가 잡아내는 '수상한 행동' 예시
보안 담당자로서 팔코를 통해 다음과 같은 상황을 즉시 알 수 있습니다.
비정상적 실행:
"웹 서버 파드 안에서 갑자기
nmap
이나
nc
(넷캣) 명령어가 실행되었습니다!"
파일 접근:
"읽기 전용이어야 할 설정 파일이 수정되었습니다!"
권한 상승:
"특정 컨테이너가 호스트의 커널 모듈을 로드하려고 시도합니다!"
4. 쉬운 비유
이미지 스캔 (어제 내용):
입국 전
'비자 심사'
어드미션 컨트롤러:
공항 입구의
'보안 검색대'
팔코 (오늘 내용):
건물 내부 곳곳에 설치된
'지능형 CCTV'
. 인가된 사람이라도 갑자기 금고를 열려고 하면 즉시 경보를 울리는 것과 같습니다.
💡 오늘의 요약
"팔코는 쿠버네티스 내부의 모든 움직임을 감시하는 '지능형 보안 카메라'다. 설정된 규칙에 어긋나는 행동을 하는 즉시 알람을 주어, 침해 사고의 골든타임을 확보하게 해준다!"
실무 활용 팁:
팔코에서 발생한 알람을 어제 배운
EFK 스택
이나 사용자님의
Slack/이메일
로 연동해두면, 24시간 내내 클러스터를 직접 들여다보지 않아도 이상 징후를 즉시 파악할 수 있습니다.
오늘 공부하신 내용 확인하셨나요? 사용자님의 **"확인완료"**가 있어야 내일 진도를 나갈 수 있습니다. (확인이 안 되면 내일은 이 내용을 다시 복습해 드릴게요!)
