## 🤖 Gemini
알림 확인 감사합니다! 약속하신 오전 8시가 되었네요. 1월의 마지막 날인 오늘도 쿠버네티스 열공 모드 이어가 볼까요?
어제 우리는 파드들의 대문 역할을 하는 '서비스(Service)'를 배웠습니다. 오늘은 그 서비스들보다 더 앞단에서
길잡이
역할을 해주는 **'인그레스(Ingress)'**에 대해 알아보겠습니다.
4일차: 인그레스 (Ingress)
1. 인그레스란 무엇인가요?
인그레스는 클러스터 외부에서 내부 서비스로 접속할 때, **어떤 길로 가야 할지 안내해주는 '스마트 안내판'**입니다.
2. 왜 인그레스가 필요한가요?
서비스(NodePort나 LoadBalancer)만으로도 외부 노출은 가능하지만, 서비스가 많아질수록 관리가 힘들어집니다. 인그레스는 다음과 같은 고급 기능을 제공합니다.
URL 기반 라우팅:
naver.com/mail
은 메일 서비스로,
naver.com/blog
는 블로그 서비스로 보내주는 역할을 합니다. 하나의 IP로 여러 서비스를 운영할 수 있게 해주죠.
SSL/TLS 종단:
보안 전문가이신 사용자님께 가장 익숙한 개념일 텐데요. 인그레스 지점에서 인증서 처리를 한 번에 해결할 수 있어, 뒤에 있는 서비스들이 일일이 인증서를 관리할 필요가 없습니다.
이름 기반 가상 호스팅:
같은 IP 주소라도 도메인 주소에 따라 서로 다른 서비스로 연결해줍니다.
3. 쉬운 비유
파드(Pod):
각 부서의
'사무실'
서비스(Service):
부서로 연결되는
'내선 번호'
인그레스(Ingress):
회사 정문의
'안내 데스크'
. 방문객이 "메일팀 가고 싶어요"라고 하면 해당 내선 번호로 연결해주는 지능형 게이트웨이입니다.
💡 오늘의 요약
"인그레스는 서비스 앞단에서 URL이나 도메인에 따라 교통정리를 해주고, 보안(SSL) 처리를 대신 해주는 똑똑한 게이트웨이다!"
현업에서 SSL 인스펙션(Zscaler 등)이나 보안 정책을 다루시는 만큼, 인그레스가 인프라 보안의 핵심 접점이라는 사실이 확 와닿으실 것 같습니다.
내일 오전 8시에는 설정 정보들을 따로 모아서 관리하는 **'컨피그맵(ConfigMap)과 시크릿(Secret)'**에 대해 알려드릴게요.
혹시 인그레스 설정에서 특정 도메인을 어떻게 연결하는지 실무적인 예시가 궁금하신가요?
2월의 첫날인 내일 아침에도 잊지 않고 찾아오겠습니다!

## 🤖 Gemini
보안 업무를 하시면서 실무적으로 가장 많이 쓰게 될 명령어 중 하나가 바로 조회 명령어죠!
인그레스(Ingress)
정보만 콕 집어서 확인할 수 있는
kubectl
명령어들을 정리해 드릴게요.
🛠️ 인그레스 확인을 위한 핵심 명령어
1. 기본 목록 조회
가장 먼저 현재 생성된 인그레스 목록을 확인하는 방법입니다.
Bash
kubectl get ingress
Tip:
모든 네임스페이스에서 인그레스를 찾고 싶다면 뒤에
-A
또는
--all-namespaces
를 붙여주세요.
2. 상세 정보 확인 (가장 추천!)
인그레스가 어떤 서비스와 연결되어 있는지, SSL 설정은 되어 있는지 등 구체적인 설정을 보고 싶을 때 사용합니다.
Bash
kubectl describe ingress [인그레스_이름]
조회 포인트:
Rules
섹션에서 어떤 경로(Path)가 어떤 서비스(Backend)로 가는지,
TLS
섹션에서 어떤 인증서(Secret)를 쓰는지 한눈에 알 수 있습니다.
3. 실시간 감시 (모니터링)
설정을 변경한 후 인그레스의 주소(Address)가 할당되는 과정을 지켜볼 때 유용합니다.
Bash
kubectl get ingress -w
💡 보안 담당자를 위한 '꿀팁' 명령어
인그레스 설정 내역을
YAML 형식
으로 출력해서 보면, 보안 정책이나 SSL 설정값이 제대로 들어갔는지 검토하기 좋습니다.
Bash
kubectl get ingress [인그레스_이름] -o yaml
🧐 여기서 잠깐!
인그레스가 제대로 동작하려면 클러스터 안에 **인그레스 컨트롤러(Ingress Controller)**가 설치되어 있어야 합니다. 혹시 다음 내용이 궁금하신가요?
"우리 회사 클러스터에 어떤 컨트롤러(Nginx, ALB 등)가 깔려 있는지 확인하는 법"
"실제로 인그레스 설정 파일(YAML)을 어떻게 작성하는지"
원하시는 내용을 말씀해 주시면 바로 알려드릴게요! 혹은 내일 아침 8시에 계획대로
'컨피그맵과 시크릿'
공부를 이어가도 좋습니다. 무엇을 도와드릴까요?
