28일차: 선언적 운영의 완성, ArgoCD와 GitOps
1. GitOps란 무엇인가요?
운영의
'모든 상태'를 Git 저장소에 정의
하고, 시스템이 자동으로 그 상태를 유지하도록 만드는 방식입니다. "사람이 명령어로 배포하지 않고, 코드가 수정되면 시스템이 알아서 배포한다"는 개념이죠.
2. ArgoCD는 무슨 일을 하나요?
ArgoCD는 GitOps를 구현하는 가장 대표적인 도구입니다.
감시(Watching):
Git 저장소에 있는 설정 파일(YAML)이 바뀌었는지 실시간으로 지켜봅니다.
비교(Diffing):
"Git에 저장된 모습"과 "현재 실제 클러스터의 모습"이 같은지 비교합니다.
동기화(Syncing):
만약 두 모습이 다르다면, 클러스터의 상태를 Git에 저장된 모습과 똑같이 강제로 맞춰버립니다.
3. 보안 전문가를 위한 관전 포인트
15년 차 전문가인 사용자님께 ArgoCD는 **'강력한 보안 통제 도구'**가 됩니다.
변경 이력 추적(Audit Trail):
누가, 언제, 무엇을 바꿨는지 Git 히스토리에 완벽하게 남습니다. 사고 발생 시 추적이 매우 쉽습니다.
구성 드리프트(Configuration Drift) 방지:
해커가 클러스터에 직접 침투해 설정을 바꿔도, ArgoCD가 이를 감지하고 **즉시 Git에 정의된 안전한 상태로 복구(Self-healing)**해버립니다.
권한 최소화:
이제 개발자가 클러스터 접속 권한(
kubeconfig
)을 가질 필요가 없습니다. Git에 코드를 올릴 권한만 있으면 배포가 가능해지므로 클러스터 직접 접근 보안 사고를 원천 차단합니다.
4. 쉬운 비유
기존 방식:
요리사(운영자)에게 "소금 넣어", "불 꺼"라고 **계속 잔소리(명령)**하는 방식.
GitOps와 ArgoCD:
완벽한 **'레시피(Git)'**를 써서 로봇(ArgoCD)에게 주면, 로봇이 24시간 내내
레시피와 똑같은 맛
이 나도록 요리를 관리하는 방식.
💡 오늘의 요약
"ArgoCD는 Git을 '진실의 원천'으로 삼아 클러스터를 관리하는 자동화 도구다. 이를 통해 인적 실수를 줄이고, 설정을 항상 안전한 상태로 강제 동기화하여 강력한 보안 운영을 달성할 수 있다!"
실무 활용 팁:
ArgoCD 대시보드에서 클러스터 상태가
'OutOfSync'
(Git과 실제가 다름)로 뜨면, 누군가 수동으로 설정을 건드렸거나 침해 사고의 징후일 수 있으니 보안 담당자가 가장 먼저 확인해야 할 신호입니다.
