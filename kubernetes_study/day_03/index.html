Kubernetes Study - Day 3

👤 User

쿠버네티스(Kubernetes) 핵심 개념 하나를 선정해서 학습하기 쉽게 설명해줘.

🤖 Gemini

알림 확인 감사합니다! 약속하신 오전 8시가 되었네요. 어제 우리는 파드를 관리하는 '디플로이먼트'에 대해 배웠습니다. 그런데 파드는 죽고 다시 살아날 때마다 IP 주소가 계속 바뀐다는 문제점이 있어요. 그래서 오늘은 이 파드들에게 변하지 않는 접속 창구를 만들어주는 '서비스(Service)'에 대해 알아보겠습니다. 3일차: 서비스 (Service) 1. 서비스란 무엇인가요? 서비스는 여러 개의 파드를 하나로 묶어 고정된 IP 주소나 이름 을 부여하는 일종의 '단일 진입점'입니다. 2. 왜 서비스가 필요한가요? 쿠버네티스에서 파드는 언제든 죽고 새로 생성될 수 있습니다. 이때마다 파드의 IP가 바뀌는데, 외부에서 바뀐 IP를 매번 알아내서 접속하는 것은 불가능에 가깝습니다. 고정된 주소: 파드가 수십 번 바뀌어도 서비스의 IP는 변하지 않습니다. 사용자는 서비스 주소로만 접속하면 됩니다. 로드 밸런싱 (Load Balancing): 서비스는 들어오는 요청을 뒤에 있는 여러 개의 파드에 골고루 나눠줍니다. 보안 장비 앞단에 배치하는 로드 밸런서와 역할이 매우 흡사하죠. 3. 주요 서비스 타입 ClusterIP (기본): 클러스터 내부에서만 접근할 수 있는 주소를 만듭니다. (내부 서버 간 통신용) NodePort: 모든 노드(서버)의 특정 포트를 개방하여 외부 접속을 허용합니다. LoadBalancer: 클라우드 환경(AWS, Azure 등)에서 외부 로드 밸런서를 자동으로 생성해 연결합니다. 💡 오늘의 요약 "파드는 계속 변하지만, 서비스는 변하지 않는 대문 역할을 한다. 사용자는 대문(Service)을 통해 안정적으로 방(Pod)으로 들어갈 수 있다!" 보안 인프라를 다뤄오셨으니, 특정 IP에 대한 접근 제어(ACL)를 설정할 때 이 '서비스' 개념이 얼마나 핵심적인지 바로 이해되실 겁니다. 내일 오전 8시에는 이 서비스들을 외부와 더 스마트하게 연결해주는 '인그레스(Ingress)'에 대해 알려드릴게요. 혹시 실제 설정 파일(YAML) 예시가 궁금하시거나, 특정 서비스 타입에 대해 더 자세히 알고 싶으신가요?
