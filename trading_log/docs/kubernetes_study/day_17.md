17일차: 클러스터의 최종 파수꾼, 어드미션 컨트롤러
1. 어드미션 컨트롤러란 무엇인가요?
사용자가
kubectl apply
같은 명령어로 쿠버네티스에 요청을 보냈을 때, 그 요청이 실제 실행되기 직전에 **'이 요청이 우리 회사의 보안 정책에 맞는가?'**를 최종적으로 검사하고 승인하는 관문입니다.
2. 왜 이게 중요한가요? (보안 전문가를 위한 핵심)
RBAC(권한 관리)이 "누가 들어올 수 있는가?"를 본다면, 어드미션 컨트롤러는 **"가져온 짐(설정) 속에 위험한 것은 없는가?"**를 봅니다.
보안 강제화:
"모든 파드는 반드시 보안 스캔을 거친 이미지만 써야 한다"거나 "루트(Root) 권한으로 실행되는 컨테이너는 생성을 거부한다"는 정책을 강제할 수 있습니다.
자동 설정 보완:
사용자가 깜빡하고 보안 설정을 안 넣었을 때, 컨트롤러가 자동으로 "보안 라벨"이나 "리소스 제한"을 추가해서 승인해줄 수도 있습니다.
3. 두 가지 핵심 단계
Mutating (변고):
들어온 요청 내용을 정책에 맞게
수정
하거나 보완합니다. (예: 보안 사이드카 컨테이너 자동 삽입)
Validating (검증):
요청 내용이 정책에 어긋나면 아예
거절
합니다. (예: 허용되지 않은 외부 저장소 이미지 사용 시 차단)
4. 쉬운 비유
RBAC:
아파트 단지에 들어올 수 있는
'출입증 검사'
어드미션 컨트롤러:
현관문 앞에서 택배 상자를 열어 **'위험물이나 반입 금지 물품이 있는지 검사'**하고, 필요하면 주의 스티커를 붙이는 보안 요원
💡 오늘의 요약
"어드미션 컨트롤러는 클러스터의 '최종 보안 게이트'다. 아무리 권한이 있는 사용자라도 보안 규칙에 어긋나는 설정을 가져오면 입구에서 컷트하여 클러스터의 안전을 유지한다!"
보안 실무에서는 **OPA(Open Policy Agent)**나
Kyverno
같은 도구를 사용하여 "우리 회사만의 전용 보안 규칙"을 이 어드미션 컨트롤러 단계에 심어놓곤 합니다.
오늘 공부하신 내용 확인하셨나요? **"확인완료"**라고 말씀해 주시면, 내일 오전 8시에 다음 진도인 **'클러스터 외부와의 통신, 로드밸런서와 노드포트'**에 대해 상세히 알려드릴게요!
확인이 안 되면 내일은 진도 없이 이 내용을 다시 복습해 드립니다. 오늘도 안전한 하루 보내세요!

🛡️ 쿠버네티스 네이티브 정책 엔진: 키르베노(Kyverno)
키베르노는 쿠버네티스 전용으로 설계된 보안 정책 관리 도구입니다. 복잡한 프로그래밍 언어를 배울 필요 없이, 우리가 익숙한
YAML 파일
만으로 보안 정책을 만들 수 있다는 게 가장 큰 장점입니다.
1. 키베르노가 하는 3가지 핵심 역할
검증 (Validate):
"이 설정은 위험해! 생성 안 돼!"라고 거절하는 역할입니다.
실무 예시:
"루트 권한으로 실행되는 파드는 차단해."
변조 (Mutate):
"사용자가 설정을 빠뜨렸네? 내가 대신 넣어줄게."라고 수정하는 역할입니다.
실무 예시:
"사용자가 리소스 제한(Limit)을 안 적었으면 자동으로 CPU 0.5개를 할당해."
생성 (Generate):
"새로운 공간(Namespace)이 생겼네? 보안 설정도 같이 만들어줄게."라고 생성하는 역할입니다.
실무 예시:
"새 네임스페이스가 생기면 기본 네트워크 보안 정책(NetworkPolicy)을 자동으로 생성해."
2. 왜 다른 도구(OPA 등)보다 키베르노인가요?
쿠버네티스 친화적:
쿠버네티스 리소스를 그대로 사용하므로 러닝 커브가 매우 낮습니다.
시각화:
정책이 잘 적용되고 있는지, 누가 정책을 어기려 했는지 보고서(Policy Report) 형태로 깔끔하게 보여줍니다.
3. 보안 전문가를 위한 '키베르노 정책' 예시 (YAML)
보안 담당자로서 이런 정책을 한 번만 배포해두면, 클러스터 전체의 보안 수준이 확 올라갑니다.
YAML
# 예: 특정 레지스트리(우리 회사 전용) 이미지만 사용하도록 강제하는 정책
apiVersion:
kyverno.io/v1
kind:
ClusterPolicy
metadata:
name:
restrict-image-registry
spec:
validationFailureAction:
Enforce
# 정책 위반 시 즉시 차단
rules:
-
name:
validate-registries
match:
resources:
kinds:
-
Pod
validate:
message:
"우리 회사 공식 레지스트리(my-reg.com) 이미지만 사용 가능합니다."
pattern:
spec:
containers:
-
image:
"my-reg.com/*"
💡 오늘의 요약
"키베르노는 보안 담당자가 원하는 보안 규칙(YAML)을 쿠버네티스 입구에 심어두는 '지능형 보안 센서'다. 이를 통해 실수를 방지하고 강력한 보안 가이드라인을 강제할 수 있다!"
