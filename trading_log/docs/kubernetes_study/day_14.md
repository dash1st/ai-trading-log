14일차: Istio와 사이드카(Sidecar)
1. Istio란 무엇인가요?
Istio는 쿠버네티스 위에 올라가는
스마트한 관리 계층
입니다. 수많은 서비스(Pod)들이 서로 통신할 때, 그 길목마다 보안 요원을 배치하여 통신을 암호화하고, 정책을 검사하고, 트래픽을 제어합니다.
2. 핵심 동작 방식: 사이드카(Sidecar) 패턴
Istio의 핵심은
사이드카
입니다. 오토바이 옆에 보조 좌석(Sidecar)을 달 듯, 우리가 만든 파드(Pod) 안에 **'통신 및 보안 전담 컨테이너(Envoy Proxy)'**를 자동으로 하나 더 끼워 넣는 방식입니다.
개발자의 편의:
개발자는 보안 코드를 짤 필요가 없습니다. 그냥 앱을 띄우면 Istio가 옆에 보안 요원(사이드카)을 딱 붙여줍니다.
보안의 일관성:
모든 파드가 동일한 보안 요원을 가지고 있으니, 전체 클러스터의 보안 정책을 한곳에서 일괄적으로 관리할 수 있습니다.
3. 보안 담당자가 Istio를 좋아하는 이유 (Zero Trust)
보안 전문가이신 사용자님께 가장 중요한 키워드는 바로 **제로 트러스트(Zero Trust)**일 것입니다.
강력한 인증(mTLS):
파드끼리 통신할 때 서로의 신분증(인증서)을 확인하고 암호화합니다. 내부 네트워크에 침투한 해커가 패킷을 훔쳐봐도 내용은 암호화되어 있어 읽을 수 없습니다.
세밀한 인가(Authorization):
"A 네임스페이스의 파드는 B 네임스페이스의 특정 DB 포트로만 접속 가능하다"는 식의 정책을 아주 쉽게 강제할 수 있습니다.
💡 오늘의 요약
"Istio는 클러스터의 모든 통신 경로에 '사이드카'라는 보안 요원을 배치한다. 이 요원들이 mTLS 암호화와 출입 통제를 대신 해줌으로써, 완벽한 제로 트러스트 보안 환경을 구축한다!"
실무 활용 팁:
실제로 구축 아파트에서 보안 업무를 하실 때 단지 내 CCTV와 출입 통제 시스템을 중앙에서 관리하는 것과 비슷합니다. 각 세대(Pod)마다 스마트 도어락(Sidecar)을 설치하고 관리실(Control Plane)에서 전체 권한을 제어하는 셈이죠.
