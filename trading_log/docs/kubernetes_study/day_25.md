25일차: 자동 확장 시스템, HPA (Horizontal Pod Autoscaler)
1. HPA란 무엇인가요?
HPA는
"부하가 늘어나면 파드(Pod)의 개수를 자동으로 늘리고, 한가해지면 다시 줄이는"
똑똑한 비서입니다.
사용자가 일일이 모니터링하며 파드 개수를 조절할 필요 없이, 설정해둔 기준(예: CPU 사용량 60%)을 넘기면 쿠버네티스가 알아서 파드를 복제(Scale-out)합니다.
2. 왜 이게 중요한가요? (보안 및 운영 관점)
가용성 보장 (Availability):
갑작스러운 접속자 폭주(DDoS 공격이나 이벤트 등) 상황에서도 서비스가 죽지 않고 버틸 수 있게 해줍니다.
비용 절감:
트래픽이 적은 밤 시간대에는 파드 수를 줄여 클라우드 자원 낭비를 막습니다.
안정적인 보안 관제:
자원 부족으로 로그 수집 파드나 보안 에이전트가 죽는 상황을 방지하여 관제의 연속성을 유지합니다.
3. 어떻게 작동하나요?
메트릭 수집:
Metrics Server
가 파드의 CPU/메모리 사용량을 실시간으로 체크합니다.
판단:
"오, 지금 CPU 사용량이 설정값인 50%를 넘어서 80%네?"라고 판단합니다.
실행:
디플로이먼트(Deployment)에 명령을 내려 파드를 하나 더 띄웁니다.
4. 쉬운 비유
파드(Pod):
은행의
'업무 창구'
HPA:
은행의
'순번 대기 시스템과 탄력 근무제'
.
대기 인원(부하)이 많아지면 숨어있던 직원들을 불러내어
비어있는 창구를 추가로 오픈
하고, 손님이 없으면 다시 업무를 정리하게 하는 것과 같습니다.
💡 오늘의 요약
"HPA는 트래픽에 따라 파드 수를 스스로 조절하는 '자동 변속기'다. 이를 통해 서비스의 중단 없는 운영과 자원의 효율적 사용을 동시에 달성할 수 있다!"
실무 활용 팁:
HPA를 잘 쓰려면 반드시 21일 차에 배웠던 **Resource Limit(자원 제한)**이 파드에 설정되어 있어야 합니다. 기준점(Limit)이 있어야 "몇 퍼센트를 쓰고 있다"는 계산이 가능하기 때문입니다.
