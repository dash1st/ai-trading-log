지금까지 파드를 만들고(Deployment), 길을 열어주고(Service, Ingress), 직접 실습하는 법까지 알아봤습니다. 오늘은 애플리케이션의 **'설정값'**과 **'비밀번호'**를 관리하는 핵심 개념인 **컨피그맵(ConfigMap)**과 **시크릿(Secret)**을 묶어서 설명해 드릴게요.
5일차: 컨피그맵(ConfigMap) & 시크릿(Secret)
보안 전문가이신 사용자님께는 이 개념이 특히 중요할 것 같습니다. 코드와 설정(비밀번호 등)을 분리하는 보안의 기본 원칙을 쿠버네티스에서 구현하는 방법이거든요.
1. 왜 따로 관리하나요?
애플리케이션 코드는 하나지만, 개발 환경(Dev)과 운영 환경(Prod)의 DB 주소나 비밀번호는 다르죠. 환경이 바뀔 때마다 코드를 다시 수정해서 빌드하는 건 비효율적이고 위험합니다. 그래서
코드는 그대로 두고, 설정값만 외부에서 주입
해주는 방식을 사용합니다.
2. 컨피그맵(ConfigMap)
역할:
일반적인 설정 정보(DB 호스트 주소, 환경 변수, 설정 파일 등)를 저장합니다.
특징:
누구나 읽을 수 있는 평문(Plain Text) 데이터입니다.
3. 시크릿(Secret)
역할:
유출되면 안 되는
민감한 정보
(비밀번호, API 키, SSH 키, 인증서)를 저장합니다.
특징:
데이터가 Base64로 인코딩되어 저장됩니다. (단, 암호화는 아니므로 실제 운영 환경에서는 추가 보안 설정이 필요합니다.)
4. 어떻게 사용하나요?
파드를 실행할 때 이 저장소에서 값을 가져오라고 지정하면, 파드 내부의
환경 변수
로 들어가거나 특정 경로의
파일
로 마운트되어 나타납니다.
