6일차: 볼륨(Volume)과 스토리지
1. 왜 별도의 저장 공간이 필요한가요?
기본적으로 쿠버네티스의 파드(Pod)는
일시적
입니다. 파드가 죽었다가 다시 살아나면, 그 안에서 작업했던 파일이나 로그는 모두 사라져 버립니다(이를 '휘발성'이라고 하죠). DB 데이터나 업로드된 이미지처럼
파드가 죽어도 사라지면 안 되는 데이터
를 보관하기 위해 외부 저장소인
볼륨
을 연결해야 합니다.
2. 핵심 개념 3가지 (이것만 기억하세요!)
PV (Persistent Volume):
실제 물리적인 저장 공간입니다. (예: 서버의 하드디스크, AWS EBS, NFS 등) 관리자가 미리 준비해둔 **'실제 창고'**라고 보시면 됩니다.
PVC (Persistent Volume Claim):
사용자가 "나 이 정도 크기의 저장 공간이 필요해!"라고 요청하는 **'주문서'**입니다.
StorageClass:
주문서가 들어오면 자동으로 창고(PV)를 생성해주는 **'자동화 시스템'**입니다.
3. 쉬운 비유
파드(Pod):
세입자
PVC (주문서):
"방 3개짜리 집을 빌리고 싶어요"라는 임대 신청서
PV (창고):
실제 지어져 있는 아파트 건물
쿠버네티스:
신청서(PVC)를 보고 비어있는 아파트(PV)를 연결해주는 중개인
💡 오늘의 요약
"파드는 언제든 사라질 수 있지만, 소중한 데이터는 PV라는 외부 창고에 저장해두어 파드가 교체되어도 데이터를 계속 유지할 수 있게 한다!"
보안 전문가 입장에서 보면, 이 볼륨 설정은
백업 및 복구(DR)
전략과 직결됩니다. 또한, 시크릿 데이터를 파일 형태로 볼륨에 마운트하여 사용하는 경우도 많으므로 보안 점검 시 볼륨 권한 설정(
readOnly
등)을 확인하는 것도 매우 중요합니다.
