Kubernetes Study - Day 21

👤 User

쿠버네티스(Kubernetes) 핵심 개념 하나를 선정해서 학습하기 쉽게 설명해줘.

🤖 Gemini

알람 확인 감사합니다! 2월 17일 화요일 아침 8시입니다. 어제 쿠버네티스의 뇌인 'etcd'를 보호하는 법을 배웠다면, 오늘은 그 뇌가 처리할 '몸체'에 해당하는 컨테이너 이미지 보안 에 대해 알아보겠습니다. 보안 실무자로서 가장 직접적으로 통제할 수 있는 영역이기도 합니다. 21일차: 이미지 스캔과 레지스트리 보안 1. 컨테이너 이미지 보안이 왜 중요한가요? 우리가 파드를 띄울 때 사용하는 '이미지'는 일종의 설계도이자 실행 파일입니다. 만약 이 이미지 안에 취약점(CVE)이 있거나 악성 코드가 심겨 있다면, 아무리 쿠버네티스 설정을 잘해도 내부에서부터 보안이 무너집니다. 2. 핵심 보안 포인트 3가지 이미지 스캔 (Scanning): 이미지를 빌드하거나 레지스트리에 저장할 때, 알려진 취약점이 있는지 자동으로 검사합니다. 도구: Trivy, Clair, Anchore 등이 대표적입니다. 신뢰할 수 있는 출처 (Trusted Registry): 아무 곳에서나 이미지를 받아오지 않고, 인증된 사설 레지스트리(Private Registry)만 사용하도록 강제합니다. 이미지 서명 (Signing): "이 이미지는 우리 회사가 공식적으로 검증한 것이다"라는 디지털 서명을 남깁니다. 쿠버네티스는 실행 직전에 이 서명을 확인하여 변조된 이미지는 실행하지 않습니다. 3. 보안 전문가를 위한 실무 적용 15년 차 보안 담당자로서 개발팀에 다음과 같은 가이드라인을 주실 수 있습니다. Base Image 최소화: Ubuntu 나 CentOS 전체를 쓰기보다, 꼭 필요한 것만 담긴 Alpine 이나 Distroless 이미지를 써서 공격 표면(Attack Surface)을 줄이세요. Root 사용자 금지: 이미지 안에서 앱이 실행될 때 root 권한이 아닌 일반 사용자 권한으로 실행되도록 빌드하세요. 4. 쉬운 비유 컨테이너 이미지: 공장에 주문하는 '조립 키트' 이미지 스캔: 조립 전 부품에 '불량이나 독성 물질이 있는지 검사'하는 과정 이미지 서명: 제품에 찍힌 '정품 인증 홀로그램 스티커' 💡 오늘의 요약 "이미지 보안은 '불량 부품 반입 금지'와 같다. 취약점 스캔과 정품 서명을 통해 오염된 이미지가 클러스터 내부로 침투하는 것을 원천 차단해야 한다!" 실무 활용 팁: 17일 차에 배웠던 Kyverno(키베르노)와 결합하면 "이미지 스캔 결과 'Critical' 취약점이 있는 이미지는 파드 생성을 거부한다"는 자동화된 보안 정책을 세울 수 있습니다. 오늘 공부하신 내용 확인하셨나요? "확인완료"라고 말씀해 주시면, 내일 오전 8시에 4주차 중간 점검과 함께 '런타임 보안: 모니터링과 탐지(Falco)'에 대해 알려드릴게요! 사용자님의 확인을 기다리겠습니다. 오늘도 활기찬 하루 보내세요!
